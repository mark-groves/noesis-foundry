# Useful Memory Requires a Managed Lifecycle

## Claim

Noesis memory should move through explicit lifecycle stages instead of treating
all notes as equally current and equally trustworthy.

## Supporting Evidence

- [[evidence-memory-lifecycle]] shows that the README defines useful memory as
  structured, reviewed, contextual, and lifecycle-aware.

## Limits

This claim is grounded in the project vision. It does not yet prove that the
proposed folder schema is sufficient for all future Noesis domains.

## Review Notes

Approved in [[review-local-first-lifecycle]] for the prototype because it
directly matches README and VISION language.
