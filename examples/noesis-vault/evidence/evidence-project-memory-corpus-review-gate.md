# Project Memory Corpus Review Gate Evidence

## Evidence

The checked-in roadmap and agent-memory chains already use review notes before
promotion, while stale-memory notes preserve superseded shortcuts without
placing their instructions into active context.

## Source Basis

- [[source-project-memory-corpus-review-governance]]

## Extraction Notes

This evidence is about the review gate between source-backed material and active
project memory. It does not require a Codex-specific schema or session metadata
to decide whether a memory item can become operational context.

## Candidate Claims

- Project-memory corpus entries should make the review gate explicit and keep
  stale shortcuts traceable outside active context.
