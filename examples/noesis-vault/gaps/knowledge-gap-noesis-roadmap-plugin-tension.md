# Noesis Roadmap Plugin-First Tension

## Gap

The vault preserves a tension between the current roadmap claim that project
memory and adapters should mature first, and the superseded plugin-first
assumption retained for traceability.

## Why It Exists

The current roadmap evidence says the Markdown plus flat YAML vault is the
durable contract and CLI, MCP, and skills are adapters over shared helpers.
The stale roadmap note preserves the older idea that a custom Obsidian plugin
should lead the next phase. The open question is how users and agents should
keep seeing this tension without reactivating stale memory as active context.
